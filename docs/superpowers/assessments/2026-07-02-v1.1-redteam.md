# TrailPrint v1.1 — Red-Team Assessment (post-P0)

**Date:** 2026-07-02 · **Scope:** `main` after PRs #4–#8 (v1.1 code-complete)
**Method:** 5 adversarial lenses (backend correctness, redundancy/dead code, operator
workflow + frontend↔backend contract, poster-beauty vs the original intent, tests/CI),
62 agents, every finding independently verified. **57 findings → 53 confirmed, 4 false
positives.** This document records the verdict; the companion PR fixes everything
fixable and this file marks what remains.

## 1. Verdict

v1.1's foundations held up: **no fabricated-terrain path, registration proven on real
ground, determinism intact, no data-corruption or crash-class defect in the render
engine.** The confirmed findings cluster in three bands:

1. **The seams between finished parts** — the accumulate-upload flow destroyed marker
   annotations; the v1.1 headline features (title, PDF) existed only at the API layer
   and were unreachable from the wizard; journey-day grouping was unguarded end-to-end.
2. **Second-order robustness** — NaN-poisoned sessions, unbounded concurrent renders,
   TTL sweeps deleting photos under live sessions, a path-traversal latent in the blob
   seam, missing upload size caps.
3. **Beauty at the margins** — playa/marsh polygons rendered as solid lakes, vinyl-flat
   water, invisible terminus pins, the watermark parked on the route corridor,
   region-name-only titles.

## 2. Fixed in the companion PR (all verified by new tests)

**Correctness (bugs):** annotation carry-forward on accumulate (labels/icons/photos
transfer to the nearest recomputed hotspot or survive verbatim); in-bounds enforcement
on the session + single-region upload branches; NaN marker moves → 422 before any
write; non-dict marker entries skipped; zoom cap enforced on **both axes** + a crop/print
**aspect-mismatch** guard (anisotropic stretch was silently renderable); render queue
bounded to one 300-dpi job at a time (`TRAILPRINT_RENDER_CONCURRENCY`); uploads sweep
skips live sessions' photo dirs; blob-key escape via sibling-prefix closed
(`commonpath`); 64 MB/file + 256 MB/batch track-upload caps; GPX `<rte>` route-only
files now ingest; day-less cross-file hotspot undercount fixed; printed mileage dedupes
duplicate uploads (same day + geometry) while honest re-travels still sum.

**Wizard (the UI finally exposes v1.1):** poster **title field** on the Frame step
(empty → region name, `-` → none); **PNG/PDF format picker** with correct download
naming; final-poll terminates on a vanished job instead of spinning forever;
**Download again** re-serves the last final without a ~80 s re-render; server error
sentences surface verbatim (no more raw JSON, no more "draw wider" advice for an
off-DEM crop); **Back to proof** after stepping back with a still-valid proof;
**Start over** reachable from every step. Verified end-to-end in a real browser
(Playwright: upload → custom title → proof → PDF download with correct magic bytes).

**Beauty:** NHD **Playa/SwampMarsh/Ice polygons no longer render as lakes** (ftype
filter in `bake_hydro`; hydro rebaked for both regions — 7 + 19 misleading polygons
dropped); lake fill at 235 alpha so the sheet's tooth ghosts through (no more vinyl
sticker); terminus pins marker-scaled (~2.6 mm — visible at viewing distance); PROOF
watermark moved off the route corridor to the upper third; `TRAILPRINT_FONT` lets the
operator drop in a licensed display face.

**Redundancy removed:** dead `detect_region`/`contains_lonlat`, dead `save_print`,
dead `spec.track_color` (serializer now tolerates unknown keys from old DB rows —
which also protects every future schema change), duplicated session-404/stamped-spec
gates unified, `rasterize` reuses the caller's parsed region config, unused imports,
47 MB of orphaned pre-blob-store finals + stale gitignore rules, `blobs` name shadowing.

**Tests/CI:** endpoint tests no longer write into the operator's live `blobs/` and
`uploads/` (isolated via env at conftest import); journey-days, `/readyz` 503, TTL=0,
queue concurrency, job-result 409/expired-404, sqlite round-trip with `track_days` +
legacy-field tolerance all covered; redundant 300-dpi renders consolidated; poll
budgets unified; region-prep logic tests now actually run (lazy fetch-stack imports +
pandas/geopandas in CI); CI gets `timeout-minutes`, superseded-run cancellation;
**uvicorn was missing from the lockfile entirely** (the pinned install couldn't serve
the app) — added.

## 3. Confirmed, deliberately NOT changed (needs Dom / product call)

- **Honey Lake renders as a solid lake.** The red-team called it a playa; the data
  says otherwise — NHD types it `LakePond` fcode 39009 (perennial), so the filter
  correctly keeps it. Overriding a source dataset by taste is a knob, not a fix. If
  you want it gone: a per-region `hydro_exclude: ["Honey Lake"]` list in `region.json`
  is a 20-minute change — say the word.
- **The 54×72 km zoom-cap floor at 18×24** (high, product): the traveled trail can
  never fill the flagship sheet — a day hike reads as a county wall-map with a gold
  thread. This is the honest cost of `native_resolution_m: 10`. Real options, all
  with costs: (a) rebuild regions on **1 m 3DEP** where available (~100× data, slow
  builds); (b) allow controlled oversampling below the cap with a quality warning;
  (c) lean into smaller print sizes for single-hike posters (9×12 exists today).
  This is the single biggest lever left on "the poster is about the journey."
- **Typography** is DejaVu-fallback until you pick/license a display face
  (`TRAILPRINT_FONT` is ready for it).
- **Marker weight is invisible** (a weight-5 place renders like a weight-1 stop) and
  the adaptive ring flips styles by terrain luminance — a taste decision on whether
  places should scale like the worn paths do.
- **Full margin-frame layout** (map inset in a paper border) — still the deliberate
  v1.2 fork it was declared to be; it changes crop-aspect semantics through the wizard.

## 4. Deferred (correct for v1-local, already roadmapped for P1/P2)

Sync proof/final still render on the event loop (P1 `run_in_threadpool`); MemoryStore
sessions never expire; single-process assumptions throughout — all named in the
roadmap's P1 section and out of scope for a local single-operator tool.
